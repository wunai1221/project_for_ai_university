# 向量資料庫規格說明（RAG ↔ Backend）

> 本文件說明 B 同學如何載入與查詢我交付的向量資料庫。

---

## 向量庫基本資訊

| 項目 | 說明 |
|------|------|
| 向量庫格式 | Chroma |
| 資料夾位置 | `/rag/vectorstore/` |
| Embedding 模型 | paraphrase-multilingual-MiniLM-L12-v2（**B 同學查詢時必須使用同一個模型**） |
| 知識類別數 | 6 類（詳見下方） |
| 資料來源 | 2 個網站、2 份 PPTX、2 份 DOCX |

---

## 六大類別

1. 鋼筋加工
2. 鋼構工程
3. 鋼材買賣及加工
4. 太陽能統包設計到送電完成
5. 餐廳推廣
6. 農產品推銷

---

## Metadata 欄位說明

每筆向量資料都附有以下 metadata，B 同學可用來過濾或顯示來源：

| 欄位 | 說明 | 範例 |
|------|------|------|
| `source_file` | 原始檔案名稱 | `"公司簡報.pptx"` |
| `source_type` | 來源格式 | `"pptx"` / `"docx"` / `"web"` |
| `page_or_section` | 頁碼或段落標題 | `"第3頁"` |
| `category` | 六大類別之一 | `"產品介紹"` |

---

## 查詢方式（程式範例說明）

B 同學載入向量庫後，查詢流程如下：

1. 用與我相同的 Embedding 模型，將使用者問題轉成向量
2. 在向量庫中搜尋相似度最高的 top-k 段落（建議 k=5）
3. 將這些段落的 `text` 欄位取出，組合成 Prompt 送給 LLM
4. （選填）可用 `category` 欄位過濾，只查詢特定類別
